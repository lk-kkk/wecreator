# V3.9 任务全生命周期增强计划

## 需求总览

在现有 7 态任务状态机基础上，插入「待付款」状态，并完善从招募→审批→执行→验收→付款→完成的完整闭环，同时增加基于任务模板节点的执行过程展示。

---

## 一、状态机改造：新增 `pending_payment`（待付款）

**现有状态链：**
```
draft → pending_review → published → in_progress → reviewing → completed → closed
```

**新状态链（8 态）：**
```
draft → pending_review → published → in_progress → reviewing → pending_payment → completed → closed
```

### 改动文件

| # | 文件 | 改动 |
|---|------|------|
| 1 | `prisma/schema.prisma` | `TaskStatus` 枚举新增 `pending_payment`；`CompanyNotificationType` 新增 `acceptance_request` |
| 2 | `prisma/migrations/2026..._v3_9.../migration.sql` | ALTER TYPE 添加新枚举值 |
| 3 | `task-status.machine.ts` | 转换表：`reviewing → [pending_payment, in_progress]`，`pending_payment → [completed]`；状态中文标签新增"待付款" |

## 二、后端 API 新增/修改

### 2.1 任务审批（招募完成后 → 进行中）

企业在任务详情中确认审批，将 `published → in_progress`。

| # | 文件 | 改动 |
|---|------|------|
| 4 | `task.controller.ts` | 新增 `POST /tasks/:id/approve`（企业审批任务） |
| 5 | `task.service.ts` | 新增 `approveTask()` 方法，校验 `published → in_progress` 转换 |

### 2.2 零工发起验收申请

零工完成任务后发起"验收申请"，任务变为 `reviewing`。

| # | 文件 | 改动 |
|---|------|------|
| 6 | `assignment.controller.ts` | 新增 `POST /worker/tasks/:assignmentId/request-acceptance`（零工发起验收申请） |
| 7 | `task.service.ts` | 新增 `requestAcceptance()` 方法：校验 `in_progress → reviewing`，创建企业通知 |

### 2.3 企业验收确认/驳回

企业在任务详情中对验收进行确认或驳回。

| # | 文件 | 改动 |
|---|------|------|
| 8 | `task.controller.ts` | 新增 `POST /tasks/:id/accept`（验收确认）和 `POST /tasks/:id/reject-acceptance`（验收驳回） |
| 9 | `task.service.ts` | 新增 `acceptTask()` 方法：`reviewing → pending_payment`，同时在 Transaction 表生成待付款记录；新增 `rejectAcceptance()` 方法：`reviewing → in_progress` 退回 |

### 2.4 支付完成 → 已完成

财务中心完成支付后，任务从 `pending_payment → completed`。

| # | 文件 | 改动 |
|---|------|------|
| 10 | `finance.controller.ts` | 新增 `POST /finance/tasks/:taskId/pay`（确认付款） |
| 11 | `finance.service.ts` | 新增 `payForTask()` 方法：创建 settlement 类型交易，更新交易状态为 completed，将任务状态从 `pending_payment → completed`，解锁资金 |

### 2.5 任务执行过程节点 API

按任务模板节点列出执行过程，聚合每个节点的状态、进度、日报、附件。

| # | 文件 | 改动 |
|---|------|------|
| 12 | `task.controller.ts` | 新增 `GET /tasks/:id/execution-nodes`（任务执行节点列表） |
| 13 | `task.service.ts` | 新增 `getExecutionNodes()` 方法：查询 checkpoints + progress_updates（按节点分组的日报）+ attachments（过程交付物），返回结构化节点数据 |

## 五、流程图

```
零工完成任务 ──→ 发起"验收申请" ──→ 企业收到消息通知
                                       │
                              ┌────────┴────────┐
                              ↓                  ↓
                        验收确认              验收驳回
                        reviewing →          reviewing →
                        pending_payment      in_progress
                              │
                              ↓
                      财务中心生成待付款记录
                              │
                              ↓
                        完成支付流程
                        pending_payment →
                        completed
                              │
                              ↓
                    财务记录变为已付款
                    任务进入"已完成"
```
