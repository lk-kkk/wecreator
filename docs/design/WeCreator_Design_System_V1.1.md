## 1. 设计原则

### 1.1 核心四原则

| 序号 | 原则 | 定义 | 落地策略 |
|------|------|------|---------|
| 01 | **信任优先 Trust First** | 涉及资金和个人信息的操作，给予充分安全感 | 充值页展示「微信支付安全」Logo；实名认证页展示隐私保护声明；结算页展示合规资质 |
| 02 | **高效引导 Guided Efficiency** | 企业发布任务的核心流程清晰快速，零工接单执行零阻力 | 5步向导+步骤进度条+草稿自动保存；零工2步完成接单 |
| 03 | **信息降噪 Focused Clarity** | 每个页面聚焦核心任务，避免信息过载 | 任务详情三栏布局各司其职；卡片折叠非核心信息；渐进披露复杂功能 |
| 04 | **双端一致 Consistent Duality** | PC端与小程序端保持品牌统一但尊重平台特性 | 统一色彩/字体/图标体系；PC端信息密度高、小程序端触控优先 |

### 1.2 设计决策优先级

```
用户安全感 > 操作效率 > 视觉美观 > 功能丰富
```

### 1.3 视觉风格定位（V1.1 新增）

基于官方视觉稿（画板1496.png）确定的视觉风格：

| 维度 | 定义 | 关键词 |
|------|------|--------|
| **色彩** | 高饱和品牌蓝主导，橙色点缀，整体明亮 | 明亮、充满活力 |
| **背景** | 浅灰页面 `#F0F0F0` + 纯白卡片，靠色差分层而非阴影 | 干净、通透 |
| **密度** | 适中偏宽松，留白充足 | 呼吸感强 |
| **圆角** | 中等圆角，卡片 8-12px | 现代、友好 |
| **阴影** | 极轻薄阴影（`0 1px 3px rgba(0,0,0,0.06)`）或无阴影 | 扁平化 |
| **图标** | 线性为主，状态标签使用彩色填充 | 清晰、辨识度高 |

### 3.7 色彩使用规则

1. **背景分层**：页面（`#F0F0F0`）→ 卡片（`#FFFFFF`）→ 蓝色强调（`#D0DCFC` 或 `#E4ECF8`）三层，靠色差而非阴影来区分层次
2. **60-30-10 法则**：60% 白/浅灰背景 + 30% 蓝色品牌色 + 10% 橙色/语义色
3. **对比度标准**：文字与背景对比度 ≥ 4.5:1（WCAG AA）
4. **蓝橙组合**：`#0858F4` + `#FC6400` 形成高能量对比，在同一视图中谨慎使用，不超过10%版面
5. **语义一致**：绿=成功/到账、红=错误/失败、橙=待处理/行动引导、青绿=执行进行中、紫=特殊等级
6. **禁止橙色文字**：`#FC6400` 不可直接用于正文色（对比度不达标），只做背景色/装饰色/图标色
