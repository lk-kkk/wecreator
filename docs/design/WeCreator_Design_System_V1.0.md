## 1. 设计原则

### 1.1 核心四原则

| 序号 | 原则 | 定义 | 落地策略 |
|------|------|------|---------|
| 01 | **信任优先 Trust First** | 涉及资金和个人信息的操作，给予充分安全感 | 充值页展示「微信支付安全」Logo；实名认证页展示隐私保护声明；结算页展示合规资质 |
| 02 | **高效引导 Guided Efficiency** | 企业发布任务的核心流程清晰快速，零工接单执行零阻力 | 5步向导 + 步骤进度条 + 草稿自动保存；零工2步完成接单 |
| 03 | **信息降噪 Focused Clarity** | 每个页面聚焦核心任务，避免信息过载 | 任务详情三栏布局各司其职；卡片折叠非核心信息；渐进披露复杂功能 |
| 04 | **双端一致 Consistent Duality** | PC端与小程序端保持品牌统一但尊重平台特性 | 统一色彩/字体/图标体系；PC端信息密度高、小程序端触控优先 |

### 1.2 设计决策优先级

```
用户安全感 > 操作效率 > 视觉美观 > 功能丰富
```

### 1.3 目标用户体验指标

| 指标 | 目标 | 测量方式 |
|------|------|---------|
| 企业任务发布时长 | < 15 分钟 | 从进入向导到确认发布的端到端时间 |
| 零工接单时长 | < 3 分钟 | 从查看邀约到确认接单 |
| 零工日报提交时长 | < 5 分钟 | 从打开进度更新到提交 |
| 首次使用学习成本 | 无需教程即可完成核心操作 | 新手可用性测试 |
| 页面信息密度 | 单屏可获取80%关键信息 | 热力图分析 |

## 2. 品牌识别

### 2.1 品牌关键词

```
专业 · 高效 · 可信赖 · 现代 · 清爽
```

### 2.2 品牌调性

| 维度 | 偏向 | 说明 |
|------|------|------|
| 情感 | 理性 > 感性 | B端产品，强调专业可靠 |
| 风格 | 简约 > 华丽 | 减少装饰元素，功能导向 |
| 色调 | 冷色 > 暖色 | 蓝色系为主，传递信任感 |
| 密度 | 适中偏紧凑 | 兼顾信息效率与呼吸感 |
| 交互 | 确定性 > 惊喜感 | 操作结果可预期，减少不确定感 |

### 2.3 Logo 规范

| 属性 | 规范 |
|------|------|
| 主Logo | 图标 + 文字横排组合，图标在左 |
| 品牌色Logo | 蓝色（#2B5BFC）用于白色/浅色背景 |
| 反白Logo | 白色版用于深色/品牌色背景 |
| 最小尺寸 | PC端：高度 ≥ 28px；小程序端：高度 ≥ 24px |
| 安全区域 | Logo四周保留 ≥ Logo高度的 50% 空白 |
| 禁止操作 | 不可拉伸变形、不可更改颜色、不可添加效果 |

### 2.4 品牌名称使用

| 场景 | 用法 |
|------|------|
| 正式文档 | WeCreator 灵活用工撮合平台 |
| 产品界面 | WeCreator |
| 简称 | WC（仅限内部代码/文档） |
| 小程序名称 | WeCreator灵工 |

### 3.5 金融专用色

| 用途 | 色值 | 说明 |
|------|------|------|
| 收入/入账 | `#10B981`（成功绿） | 充值成功、结算到账 |
| 支出/扣款 | `#EF4444`（错误红） | 锁定资金、提现扣款 |
| 冻结中 | `#F59E0B`（警告橙） | 锁定中资金、待验收冻结 |
| 金额数字 | `#111827`（文字主色） | 余额展示、流水金额 |
| 金额单位 | `#6B7280`（辅助灰） | ¥符号、元 |

### 3.6 色彩使用规则

1. **60-30-10 法则**：60% 白/灰背景 + 30% 蓝色品牌色 + 10% 语义色
2. **对比度标准**：文字与背景对比度 ≥ 4.5:1（WCAG AA 标准）
3. **色彩数量**：单个页面使用不超过 5 种色彩（含中性色）
4. **语义一致**：同一颜色在全产品范围内含义一致，绿=正向、红=负向、橙=警示
5. **禁止裸色块**：大面积色块必须搭配文字或图标，不得单独使用品牌色作为装饰

## 4. 字体排版

### 4.1 字体栈

#### PC端（企业端后台）
```css
font-family: -apple-system, BlinkMacSystemFont, 'Segoe UI', 'PingFang SC', 'Hiragino Sans GB', 'Microsoft YaHei', 'Helvetica Neue', sans-serif;
```

#### 数字/金额专用
```css
font-family: 'DIN Pro', 'DIN Alternate', -apple-system, BlinkMacSystemFont, sans-serif;
```

#### 小程序端
```css
/* 微信小程序默认字体栈，无需额外配置 */
/* iOS: PingFang SC / Android: 思源黑体 */
```

### 4.4 排版规则

| 规则 | 说明 |
|------|------|
| 标题层级 | 同一页面最多使用3级标题（H1→H2→H3） |

| 对齐方式 | 左对齐为主，金额右对齐，状态标签居中 |

| 百分比格式 | 一位小数：`85.0%` |
| 日期格式 | PC端：`2026-04-17`；小程序端：`04月17日` |
| 空数据 | 使用 `—` 而非空白 |

### 6.4 Tab图标规范（小程序端）

| 状态 | 规格 | 颜色 |
|------|------|------|
| 未选中 | 24×24px PNG / SVG | `#9CA3AF` |
| 选中 | 24×24px PNG / SVG | `#2B5BFC`（品牌蓝） |

**Tab图标设计要求：**
- 4个Tab：任务(task)、消息(message)、主页(profile)、我的(mine)
- 图标风格：2px线性，圆角末端，透明底

## 7. 基础组件规范

### 7.1 按钮 (Button)

#### 7.1.1 按钮类型

| 类型 | 样式 | 使用场景 | 优先级 |
|------|------|---------|--------|
| **主按钮 Primary** | `#2B5BFC` 填充 + 白字 | 页面核心操作（每页最多1个） | 最高 |
| **次按钮 Secondary** | `#2B5BFC` 边框 + 蓝字 + 白底 | 辅助操作（取消、查看详情） | 高 |
| **默认按钮 Default** | `#E5E7EB` 边框 + 深灰字 + 白底 | 常规操作 | 中 |
| **危险按钮 Danger** | `#EF4444` 填充 + 白字 | 关闭任务、删除操作 | 中 |
| **危险次按钮** | `#EF4444` 边框 + 红字 + 白底 | 次要危险操作 | 低 |
| **文字按钮 Text** | 蓝色文字 + 无边框无背景 | 链接式操作 | 低 |
| **图标按钮 Icon** | 纯图标 + 透明背景 | 工具栏操作 | 低 |

#### 7.1.2 按钮尺寸

| 尺寸 | PC端 | 小程序端 |
|------|------|---------|
| 大 (Large) | H 44px · Padding 0 24px · Font 16px | H 48px · Font 16px |
| 默认 (Default) | H 36px · Padding 0 16px · Font 14px | H 44px · Font 14px |
| 小 (Small) | H 28px · Padding 0 12px · Font 12px | H 36px · Font 13px |

#### 7.1.3 按钮状态

| 状态 | 视觉变化 |
|------|---------|
| Default | 正常样式 |
| Hover | 亮度+10%，PC端鼠标指针变为pointer |
| Active / Pressed | 亮度-10%，轻微缩放 `scale(0.98)` |
| Disabled | 透明度50%，鼠标变为not-allowed |
| Loading | 左侧显示旋转加载图标，文字保留 |

#### 7.1.4 按钮组合规则

- 主按钮 + 次按钮并排时，主按钮在**右侧**（PC端）
- 小程序端底部操作栏：主按钮在右，次按钮在左
- 按钮间距：12px
- 一行最多3个按钮
- 小程序端按钮最小可点击区域：44×44px（含padding）
- 危险操作按钮必须配合二次确认弹窗

### 7.2 输入框 (Input)

#### 7.2.1 基础样式

| 属性 | PC端 | 小程序端 |

### 10.1 加载状态

| 类型 | 使用场景 | 视觉表现 |
|------|---------|---------|
| **骨架屏 Skeleton** | 页面首次加载 | 灰色色块模拟内容结构，微动画闪烁 |
| **旋转加载 Spin** | 按钮操作等待 | 品牌蓝旋转图标，带文字提示 |
| **进度条 Progress** | 文件上传 | 线性进度条 + 百分比 |
| **下拉刷新** | 小程序端列表 | 品牌蓝下拉指示器 |
| **上拉加载更多** | 列表分页 | 底部旋转图标 + "加载中..." |
| **全屏加载** | 页面跳转/重大操作 | 遮罩 + 居中旋转图标 |

**骨架屏规则：**
- 闪烁动画：从 `#E5E7EB` → `#F3F4F6` → `#E5E7EB` 循环，1.5s 周期
- 圆形区域（头像）保持圆形
- 文本区域使用圆角矩形（H 16px，圆角 4px）
- 显示时长：若加载 < 300ms 则不展示骨架屏，避免闪烁

### 10.2 操作反馈

| 操作 | 反馈方式 | 反馈内容 |
|------|---------|---------|
| 表单提交成功 | Toast 成功 | "提交成功" / "保存成功" |
| 表单提交失败 | Toast 错误 + 字段标红 | "请检查标红字段" |
| 网络异常 | 全屏异常页 | 插画 + "网络开小差了" + [重试] |
| 服务器错误 | 全屏异常页 | 插画 + "服务暂时不可用" + [重试] |
| 删除操作 | 二次确认弹窗 | "确认删除？此操作不可撤销" |
| 资金操作 | 加载中弹窗 + 结果弹窗 | "支付处理中..." → "充值成功" |
| 复制成功 | Toast 成功 | "已复制到剪贴板" |
| 草稿自动保存 | 顶部浮动提示 | "草稿已自动保存 16:30" |

### 10.3 表单校验

| 校验时机 | 行为 |
|---------|------|
| 失焦校验 (onBlur) | 用户离开输入框时立即校验，错误实时展示 |
| 提交校验 (onSubmit) | 点击提交时全量校验，滚动到第一个错误字段 |
| 实时校验 (onChange) | 仅限特殊字段（如密码强度、字数限制） |

**校验提示样式：**
- 位置：输入框正下方，无间距
- 文字：12px · `#EF4444` · Regular

- 输入框：边框变为 `#EF4444` + 外发光 `0 0 0 2px #FEE2E2`

## 11. 动效规范

### 11.1 动效原则

| 原则 | 说明 |
|------|------|
| **有意义** | 每个动效都有明确目的（引导、反馈、过渡），不做纯装饰动效 |
| **克制** | 动效时长控制在 150-400ms，不拖慢操作节奏 |
| **一致** | 相同类型交互使用相同动效模式 |
| **可关闭** | 尊重系统「减弱动态效果」设置 |

### 11.2 缓动函数

| Token | 值 | 用途 |
|-------|------|------|
| `--ease-in-out` | `cubic-bezier(0.4, 0, 0.2, 1)` | 默认缓动，页面切换 |
| `--ease-out` | `cubic-bezier(0, 0, 0.2, 1)` | 元素进入、弹窗出现 |
| `--ease-in` | `cubic-bezier(0.4, 0, 1, 1)` | 元素退出、弹窗关闭 |
| `--ease-bounce` | `cubic-bezier(0.34, 1.56, 0.64, 1)` | 轻微弹性效果（成功动画） |

### 11.3 标准动效时长

| Token | 值 | 用途 |
|-------|------|------|
| `--duration-fast` | 150ms | Hover 颜色变化、Toggle |
| `--duration-normal` | 250ms | 弹窗出入、折叠展开 |
| `--duration-slow` | 400ms | 页面切换、大型动画 |

### 11.4 业务动效清单

| 场景 | 动效类型 | 时长 | 缓动 |
|------|---------|------|------|
| 侧边栏展开/折叠 | 宽度过渡 | 250ms | ease-in-out |
| 弹窗出现 | 缩放(0.9→1) + 透明度(0→1) | 200ms | ease-out |
| 弹窗关闭 | 缩放(1→0.95) + 透明度(1→0) | 150ms | ease-in |
| Toast 出现 | 从上方滑入 + 透明度 | 300ms | ease-out |
| 下拉菜单展开 | 从上到下展开 + 透明度 | 200ms | ease-out |
| Tab 切换 | 下划线滑动 | 200ms | ease-in-out |
| 卡片Hover | 阴影增强 | 150ms | ease-out |
| 进度条增长 | 宽度过渡 | 600ms | ease-in-out |
| 步骤完成 | 勾号描线动画 | 300ms | ease-out |
| 数字变化 | 数字滚动（计数器） | 800ms | ease-out |
| 小程序页面转场 | 右侧滑入（系统默认） | 300ms | 系统默认 |
| 底部弹窗 | 从底部上滑 | 300ms | ease-out |
| 红包到账 | 金额放大 + 绿色闪烁 | 500ms | ease-bounce |
