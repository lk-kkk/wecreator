# WeCreator 架构决策记录 (ADR)

> 记录项目中的关键架构决策及其理由，供团队成员和协同智能体参考。

---

## ADR-001: 后端框架选型 NestJS

- **日期**: 2026-04-16
- **状态**: 已采纳
- **决策**: 使用 NestJS 11 + TypeScript
- **理由**: 模块化架构天然适配多Agent协同开发；内置依赖注入减少耦合；Swagger自动生成API文档
- **替代方案**: Express + 手动分层 → 缺少约束，多人开发易混乱

## ADR-002: ORM 选型 Prisma 6

- **日期**: 2026-04-16
- **状态**: 已采纳（从Prisma 7降级）
- **决策**: 使用 Prisma 6.19.3
- **理由**: Prisma 7 要求 driver adapter，但截至2026-04 尚无官方 `@prisma/adapter-mysql`；Prisma 6 使用内置engine，MySQL支持稳定
- **迁移计划**: 当 Prisma 7 发布 MySQL adapter 后评估升级

## ADR-003: 数据库三件套

- **日期**: 2026-04-16
- **状态**: 已采纳
- **决策**: MySQL 8.0（主库） + Redis 7（缓存/锁） + MongoDB 6（消息）
- **理由**:
  - MySQL: 事务性业务数据（企业/任务/财务），需要ACID
  - Redis: 分布式锁（零工钱包提现）+ JWT黑名单 + 会话缓存
  - MongoDB: IM消息存储，文档结构灵活，读写性能高

## ADR-004: 资金安全双锁机制

- **日期**: 2026-04-16
- **状态**: 已采纳
- **决策**:
  - 企业余额: **乐观锁**（version字段CAS更新）
  - 零工钱包: **Redis分布式锁**（`wallet:lock:{workerId}`，TTL 10s）
- **理由**: 企业余额并发低（企业操作员少），乐观锁性能好；零工提现可能高并发（多设备操作），需强一致性

## ADR-005: 结算合规通道

- **日期**: 2026-04-16
- **状态**: 待实施
- **决策**: 接入第三方合规结算通道（薪太软/用友薪福社）
- **理由**: 企业→零工直接转账属于劳务报酬，需代扣代缴个税；平台不具备代扣资质
- **前置条件**: 签约需3-4周，Phase 0 并行启动

## ADR-006: 小程序分包策略

- **日期**: 2026-04-16
- **状态**: 已采纳
- **决策**: 4个Tab主页放主包，7个功能页放4个分包
- **理由**: 微信主包限制2MB；分包按需加载，冷启动目标 ≤3s
- **分包**: task / wallet / auth / profile

## ADR-007: Monorepo + pnpm workspace

- **日期**: 2026-04-16
- **状态**: 已采纳
- **决策**: 单仓库管理 backend + pc-admin + worker-mp + shared
- **理由**: 共享类型包 `@wecreator/shared` 确保前后端类型一致；统一CI/CD流水线

## ADR-008: 12角色Agent协同架构

- **日期**: 2026-04-16
- **状态**: 已采纳
- **决策**: 12个Agent角色（R0~R11），通过SKILL.md定义职责边界
- **理由**: 每个Agent有明确的代码管辖范围和接口契约，减少冲突；R0 Orchestrator负责仲裁

## ADR-009: JWT双Token机制

- **日期**: 2026-04-16
- **状态**: 已采纳
- **决策**: access_token 2h + refresh_token 7d
- **理由**: 短期access降低泄露风险；长期refresh减少重新登录频率；企业端和零工端统一机制

## ADR-010: 敏感字段AES-256加密

- **日期**: 2026-04-16
- **状态**: 已采纳
- **决策**: 手机号、身份证号使用AES-256-CBC加密存储
- **理由**: 个人信息保护法要求；密钥从环境变量读取，不硬编码
